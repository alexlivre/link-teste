# Regra de ESLint

- Sempre usar **ESLint** em qualquer código JavaScript/TypeScript do projeto.
- Antes de criar uma configuração nova, **verificar se já არსებობს alguma regra ou configuração de ESLint** no repositório.
- Se já existir configuração, **respeitar e seguir as regras existentes**.
- Se **não existir ESLint**, criar a configuração seguindo as **melhores práticas** do projeto.
- Sempre **testar se está tudo funcionando** após criar ou alterar a configuração.
- Sempre **corrigir quaisquer erros** encontrados no ESLint antes de considerar a tarefa concluída.
- Garantir que o código fique consistente, limpo e compatível com a base existente.
- Se houver conflito entre regras, priorizar a configuração já existente do projeto, ajustando apenas o necessário.
