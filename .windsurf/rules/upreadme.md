# Atualização de Documentação e Roadmap

## Sincronização do README
- **Gatilho de Documentação**: Sempre que o Cascade realizar a **adição de novos recursos** ou concluir o desenvolvimento de uma **fase**, o arquivo `README.md` deve ser atualizado obrigatoriamente.
- **Conteúdo da Atualização**:
    - Listar o novo recurso na seção de funcionalidades.
    - Atualizar o status da "Fase" correspondente (ex: de "Em progresso" para "Concluído" ou adicionar a próxima fase planejada).
    - Refletir qualquer alteração necessária em requisitos ou instruções de instalação que o novo recurso exija.

## Validação de Entrega
- Uma tarefa só será considerada finalizada se o código estiver funcional **e** a documentação no `README.md` refletir o estado atual do projeto.

<guidelines>
- Mantenha o tom da documentação profissional e conciso.
- Informe ao usuário após a atualização: "Recurso/Fase finalizado e documentado no README.md."
</guidelines>
